# WTODO (Weekly Todo)

## デモ
このアプリケーションは GitHub Pages で公開されており、以下のURLから直接試すことができます：
👉 **[WTODO デモサイト](https://mareku.github.io/weekly-todo/)**

WTODOは、1ファイルのHTMLで動作するシンプルかつ高機能な週間タスク管理アプリケーション（SPA）です。
美しいダークテーマと直感的な操作性、キーボードショートカットを備え、ローカル環境で軽快に動作します。

## 主な特徴

- **1ファイル構成**: サーバー不要で、`index.html` をブラウザで開くだけで今すぐ利用可能です。
- **PWA (Progressive Web App) 対応**: モバイルやPCのホーム画面・デスクトップにインストールしてスタンドアロンアプリのように使用可能で、サービスワーカー（`sw.js`）によるキャッシュによりオフライン環境でも動作します。
- **週間タスク管理**: 曜日ごとのタスクを一覧でき、週全体の進捗率（プログレスバー）を視覚的に把握できます。
- **連続達成記録 (Streak)**: タスクをすべて完了した日が続くと、ストリークカウント（🔥 〇日連続）が記録されモチベーションを維持できます。
- **キャリーオーバー機能**: 前週やり残した未完了タスクがある場合、月曜日にワンクリックで一括引き継ぎ（キャリーオーバー）が可能です。
- **繰り返しタスク**: 指定した曜日に自動でタスクを追加する繰り返し設定が可能です。
- **回数タスク (Frequency Tasks)**: 特定の曜日ではなく、「週に3回筋トレ」「月に1回読書」のように、期間（週次・月次・年次）ごとの目標回数を設定し、進捗をカウントするタスク管理機能です。
- **締切タスク (Deadline)**: 期限付きのタスクを管理でき、残り日数や超過日数がバッジで分かりやすく表示されます。また、期限日の週へワンクリックでジャンプできます。
- **期間タスク (Period)**: 開始日と終了日を設定し、進捗バーと残り日数でタスクの進行度を視覚化できます。
- **ボトムナビゲーション (Bottom Navigation)**: 画面下部にナビゲーションバーが常駐し、各セクション（回数タスク、締切タスク、期間タスク、今週のタスク）へ素早くスクロール移動できます。未完了タスクの数をバッジでリアルタイムに通知します。
- **Firebaseリアルタイム同期**: Googleログインを利用し、PC・スマホなどの複数端末間でタスクデータをリアルタイムに自動同期します。
- **オフラインファースト**: ネットワーク切断時もローカルストレージにより動作を継続し、オンライン復帰時に保留中の変更がFirebaseへ自動で送信されます。
- **データ管理**: データはブラウザのローカルストレージ（`wtodo_v4`）に自動保存されます。JSONファイル形式でのエクスポートおよびインポートにも対応しています。

## 使い方

1. `index.html` をブラウザ（Chrome、Edge、Safariなど）で開きます。
2. 画面下部または曜日カードの入力欄からタスクを追加します。
3. タスクを完了したらチェックボックスをオンにします。

### Firebase同期のセットアップ

Firebase同期を使うには、まずFirebaseプロジェクトを作成して設定値を取得する必要があります。

#### 1. Firebaseプロジェクト作成

1. [firebase.google.com](https://firebase.google.com) にアクセス
2. 「コンソールに移動」→「プロジェクトを追加」
3. プロジェクト名を入力（例：`wtodo-sync`）
4. Google アナリティクスは「無効」でOK
5. 「プロジェクトを作成」

#### 2. Realtime Database を有効化

1. 左メニュー「構築」→「Realtime Database」
2. 「データベースを作成」
3. ロケーションは **asia-southeast1（シンガポール）** が日本から近くて速い
4. セキュリティルールは「**テストモードで開始**」を選択（後でルールを変更する）

#### 3. Authentication を有効化

1. 左メニュー「構築」→「Authentication」
2. 「始める」
3. 「Sign-in method」タブ →「Google」を有効化
4. プロジェクトのサポートメールを設定して「保存」

#### 4. 承認済みドメインを追加

1. Authentication →「Settings」タブ →「承認済みドメイン」
2. 「ドメインを追加」で GitHub Pages のURL を追加
   ```
   あなたのGitHubユーザー名.github.io
   ```

#### 5. セキュリティルールを設定

Realtime Database →「ルール」タブで以下に書き換えて「公開」：

```json
{
  "rules": {
    "users": {
      "$uid": {
        ".read":  "$uid === auth.uid",
        ".write": "$uid === auth.uid"
      }
    }
  }
}
```

#### 6. 設定値を取得

1. Firebaseコンソール左上の歯車 ⚙️ →「プロジェクトの設定」
2. 「全般」タブ → 下にスクロール →「マイアプリ」
3. 「ウェブアプリを追加」（`</>`アイコン）
4. アプリのニックネームを入力（例：`wtodo-web`）
5. 「アプリを登録」するとこのような設定値が表示される：

```js
const firebaseConfig = {
  apiKey:      "AIzaSy...",        // ← API Key
  authDomain:  "wtodo-sync.firebaseapp.com",
  databaseURL: "https://wtodo-sync-default-rtdb.asia-southeast1.firebasedatabase.app",
  projectId:   "wtodo-sync",       // ← Project ID
};
```

#### 7. アプリに設定を入力

WTODOアプリで：

1. ボトムナビ「⚙️ 設定」を開く
2. 「Firebase同期」を選択
3. 取得した値を入力：
   - **API Key** → `apiKey` の値
   - **Project ID** → `projectId` の値
   - **Database URL** → `databaseURL` の値
4. 「設定を保存」
5. 「Googleでログイン」をタップ
6. Googleアカウントを選択してログイン
7. ヘッダーに「✅ 同期済み」が表示されれば完了

#### 8. 別端末でも同じ設定を行う

PCとAndroid両方で手順7を実施すれば、同じGoogleアカウントでログインした端末間でデータが自動同期されます。

### キーボードショートカット

画面上で素早く操作するためのショートカットキーが用意されています。

| キー | アクション |
| :--- | :--- |
| `n` | 今日のカードに新規タスクを追加（フォーカス） |
| `d` | 締切タスクを追加するモーダルを開く |
| `p` | 期間タスクを追加するモーダルを開く |
| `←` / `→` | 前の週 / 次の週へ移動 |
| `?` | ショートカットキーのヘルプ表示のトグル |
| `Escape` | 入力フォーカスを外す / モーダルを閉じる |

※ タスクや締切項目をタップすることで、その場でテキストを編集することができます。

## 技術仕様

- **フロントエンド**: HTML5, CSS3 (Vanilla CSS, ダークモード・レスポンシブ対応), Vanilla JS (ES6+)
- **フォント**: [Google Fonts](https://fonts.google.com/) から `Inter` および `JetBrains Mono` を読み込んで使用しています。
- **外部ライブラリ**: Firebase SDK (App, Database, Auth) をCDN経由で非同期ロードして使用しています。
- **ストレージ**: ブラウザの `localStorage` (キー名: `wtodo_v4`) および Firebase Realtime Database（ログイン時）のハイブリッド構成
- **PWA構成**: `manifest.json` によるWebアプリマニフェスト、および `sw.js` (Service Worker) によるオフライン動作とリソースキャッシュ
